# 聊天机器人用户体验设计检查清单

## 本质
一套系统化的设计原则与检查清单，用于将聊天机器人从生硬的脚本驱动工具，转变为能有效解决问题、提升用户满意度和商业价值的对话伙伴。

## 原理
**推断原理**：其有效性基于以下心理学与交互设计原理：
1.  **认知负荷理论**：通过按钮、清晰指引和结构化流程，减少用户思考和决策的负担，提升交互效率。
2.  **期望确认理论**：通过明确告知机器人能力和范围，管理用户预期，避免因能力不匹配导致的失望和挫败感。
3.  **对话原则（格莱斯合作原则）**：要求对话（人机交互）应提供适量、相关、清晰、真实的信息。好的聊天机器人设计遵循这些原则，使对话自然流畅。
4.  **容错设计**：承认用户会偏离预设路径，并提供清晰的恢复机制（如重新表达、转人工），这符合行为经济学中的“损失厌恶”心理，避免用户因一次失败体验而彻底放弃。

## 案例
**参考案例**：
*   **背景**：一家中型电商公司，希望用聊天机器人处理30%的售前咨询，减轻客服压力。
*   **做法**：
    1.  **明确范围**：在欢迎语中声明“我是购物助手小E，可以帮您查询订单、推荐商品或解答常见问题”。
    2.  **混合输入**：首先提供“订单查询”、“商品推荐”、“联系客服”三个按钮选项，同时也允许自由输入。
    3.  **设计恢复路径**：当用户问题超出范围时，回复“我暂时还不会处理这个问题，您可以点击‘联系客服’按钮，或尝试问我关于订单和商品的问题。”
    4.  **原型测试**：使用“绿野仙踪法”，由客服人员在后台模拟机器人回复，测试了50名真实用户，发现用户常问“物流到哪了”，于是优化了订单查询的对话流程。
*   **结果**：上线后，机器人成功承接了35%的常见咨询，用户满意度（CSAT）达4.2/5，客服工单量减少25%。

## 行动
1.  **第一步（今天就能做）**：**明确范围与欢迎语**。为你的聊天机器人写一段开场白，必须包含：① 它是机器人；② 它能做什么（1-3项核心功能）；③ 它不能做什么（明确限制）；④ 一个简单的使用示例（如“您可以问我‘我的订单状态’或点击下方按钮”）。
2.  **第二步**：**绘制核心对话流程图**。选择一个最常见的用户任务（如“查询订单”），用纸笔或工具（如Figma、Miro）画出“理想路径”（用户完美配合的步骤），并至少标出2个用户可能“迷路”的点（如输入了订单号格式错误、中途询问其他问题），为每个点设计一句友好的恢复引导语。
3.  **第三步**：**进行一次简易原型测试**。邀请一位同事或朋友，使用你设计的开场白和流程图，通过微信或任何即时通讯工具，由你扮演机器人（“绿野仙踪法”）与测试者进行模拟对话。记录下测试者犹豫、困惑或使用非常规说法的时刻，这些就是你需要优化的关键点。

## 边界
*   ✅ **适用场景**：设计新的聊天机器人；优化现有机器人交互体验；处理以任务完成（如查询、预订、客服）为核心的对话场景；需要平衡自动化和人工服务的产品。
*   ❌ **不适用场景**：以开放域、闲聊为核心目标的AI伴侣（如Replika）；完全无需引导、追求极致自由对话的AI研究项目；交互以复杂图形界面为主、对话仅为辅助功能的应用。
*   ⚠️ **注意事项**：设计原则需与品牌调性一致（如金融类机器人应严谨，零售类可稍活泼）；过度拟人化可能引发不切实际的用户期望；必须将无障碍设计（如屏幕阅读器兼容）纳入考量，而非事后补充。
