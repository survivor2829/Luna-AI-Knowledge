# 106个认知偏差与UX设计原理

> 类型：产品设计
> 核心学科：认知心理学 / 行为经济学 / 用户体验

## 核心问题

**用户为什么做出"不理性"的决策？设计如何顺应这些规律？**

Growth.Design将106个认知偏差按用户决策周期分为4大类，为设计师提供了系统化的心理学工具箱。

## 底层框架

### 决策周期模型

```
🙈 信息 → 🔮 意义 → ⏰ 时间 → 💾 记忆
   ↓         ↓         ↓         ↓
 如何筛选   如何解读   如何决策   如何记住
```

每个阶段都有对应的认知偏差影响用户行为。

---

## 第一阶段：信息处理

用户如何筛选海量信息？

### 希克定律 (Hick's Law)

**原理**：选项越多，决策越难

```
2个选项 → 快速决策
10个选项 → 决策瘫痪
```

**设计应用**：
- 减少表单字段
- 限制导航选项
- 渐进式披露功能

### 认知负荷 (Cognitive Load)

**原理**：大脑处理能力有限，超载导致放弃

**设计应用**：
- 简化界面元素
- 分步骤完成任务
- 使用熟悉的模式

### 渐进披露 (Progressive Disclosure)

**原理**：逐步展示比一次性展示更易接受

**设计应用**：
- "显示更多"按钮
- 分阶段引导流程
- 折叠高级选项

### 菲茨定律 (Fitts's Law)

**原理**：大的、近的目标更容易点击

**设计应用**：
- 主要按钮要大
- 重要操作放在容易触达的位置
- 移动端按钮至少44px

## 第二阶段：意义建构

用户如何解读信息？

### 锚定效应 (Anchoring Bias)

**原理**：第一个看到的信息成为判断基准

```
原价 ¥999 → 现价 ¥299
     ↑
   锚点（让¥299显得便宜）
```

**设计应用**：
- 先展示原价再展示折扣价
- 定价页先展示高价方案
- 对比表中突出差异

### 框架效应 (Framing Effect)

**原理**：同一信息，不同表述产生不同感知

| 负面框架 | 正面框架 |
|----------|----------|
| "10%失败率" | "90%成功率" |
| "限制访问" | "专属权限" |

**设计应用**：
- 用积极语言描述功能
- 强调获得而非避免

### 社会认同 (Social Proof)

**原理**：不确定时参考他人行为

**类型**：
- 用户评价和评分
- "X人正在浏览"
- 客户logo墙
- 行业认证

### 稀缺效应 (Scarcity Principle)

**原理**：供应有限时感知价值增加

```
"仅剩3件" → 激活损失厌恶
"限时优惠" → 制造紧迫感
```

**设计注意**：必须真实，虚假稀缺损害信任

## 第三阶段：时间压力

时间如何影响决策？

### 默认偏好 (Default Bias)

**原理**：人们倾向于接受预设选项

**设计应用**：
- 推荐方案设为默认
- 预填合理的表单值
- 订阅默认勾选（需合规）

### 损失厌恶 (Loss Aversion)

**原理**：失去的痛苦是获得快乐的2倍

```
"不要错过" > "立即获得"
"你将失去" > "你可以得到"
```

**设计应用**：
- 强调不行动的代价
- 试用期结束提醒
- 购物车过期提示

### 承诺一致性 (Commitment & Consistency)

**原理**：人会坚持与先前承诺一致的行为

**设计应用**：
- 小承诺引导大行动
- 进度条显示已完成部分
- "您已经完成了80%"

### 投入循环 (Investment Loops)

**原理**：投入越多，越不愿放弃

**设计应用**：
- 引导用户完善个人资料
- 创建内容后的沉没成本
- 积分/等级系统

## 第四阶段：记忆形成

用户记住什么？

### 峰终定律 (Peak-End Rule)

**原理**：体验评价取决于最高峰和结尾

```
整体体验评价 = (峰值体验 + 结尾体验) / 2
```

**设计应用**：
- 设计"惊喜时刻"
- 确保流程有好的结尾
- 成功页面要有仪式感

### 蔡格尼克效应 (Zeigarnik Effect)

**原理**：未完成任务比已完成任务记忆更深

**设计应用**：
- 保存未完成的购物车
- 进度条提示还剩多少
- 任务清单未完成提醒

### 组块化 (Chunking)

**原理**：分组信息更容易记忆

**设计应用**：
- 电话号码分段显示
- 长表单分成多步
- 卡片式信息展示

### 禀赋效应 (Endowment Effect)

**原理**：拥有（或感觉拥有）后价值感知提升

**设计应用**：
- 免费试用期
- 个性化定制体验
- "您的专属方案"

---

## 伦理边界

理解这些原理的目的是**帮助**用户，而非**操控**用户。

| 正当应用 | 暗黑模式 |
|----------|----------|
| 简化复杂决策 | 隐藏重要信息 |
| 真实的稀缺提示 | 虚假的紧迫感 |
| 帮助用户完成目标 | 诱导用户违背意愿 |

**设计者的责任**：利用心理学原理优化体验，同时尊重用户自主权。
