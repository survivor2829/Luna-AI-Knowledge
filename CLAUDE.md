# Luna AI知识库平台 - 项目规范

## 一、平台定位

我们是一个网站爬取+内容整理平台，生产可投喂给Coze智能体的高质量知识库。

**核心价值：弥补Coze/豆包原生能力做不到的事情。**

| Coze已有 | 我们补位 |
|----------|----------|
| 联网搜索 | 深度整理的专业知识 |
| 基础工作流 | 原理+案例融合 |
| 通用问答 | 可落地执行的内容 |
| - | 让用户"用了还想用" |

## 二、统一内容格式

所有爬取的内容，无论原网站是只有原理还是只有案例，都统一整理成以下格式：

```markdown
# [主题]

## 本质
[一句话说清楚这是什么，解决什么问题]

## 原理
[底层逻辑/心理学/商业原理，为什么有效]

## 案例
- 背景：[公司/场景/规模]
- 做法：[具体怎么做的]
- 结果：[量化数据]

## 行动
1. [第一步]
2. [第二步]
3. [第三步]

## 边界
[什么情况适用/什么情况不适用]
```

## 三、内容处理规则

| 原始内容 | 处理方式 |
|----------|----------|
| 只有原理没案例 | 用DeepSeek生成示例场景，标注"参考案例" |
| 只有案例没原理 | 用DeepSeek提炼底层逻辑，归纳"为什么有效" |
| 两者都有 | 直接整合成统一格式 |
| 内容太散太长 | 提炼核心要点，保留可执行部分 |

## 四、内容设计原则（参考Hook模型）

- **降低行动门槛**：每个知识点都有"立即可做"的下一步
- **提供可变奖励**：内容包含意外收获（新视角/新工具/新案例）
- **避免废话**：具体、可量化、有时间节点
- **引导深入**：文末有相关推荐，形成知识网络

## 五、质量检验标准

每篇文档必须满足：

- [ ] 有"一句话本质"（3秒能懂）
- [ ] 有"为什么有效"（原理支撑）
- [ ] 有"谁做过"（案例证明）
- [ ] 有"我能做什么"（可执行步骤）
- [ ] 有"什么情况别用"（边界条件）

## 六、爬取优先级

| 优先级 | 内容类型 |
|--------|----------|
| P0 | 带数据的真实案例 |
| P1 | 可执行的方法论/SOP |
| P2 | 心理学/行为经济学原理 |
| P3 | 行业趋势/新工具 |

## 八、一句话总结

把互联网优质内容整理成"本质+原理+案例+行动+边界"五合一格式，让Coze智能体投喂后能给出可落地、不废话、让人想继续用的回答。
