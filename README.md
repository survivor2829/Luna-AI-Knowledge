## 使用指南

### 我有一个具体场景，想知道怎么做

1. 打开 `03-索引/按场景索引.md`
2. 找到对应场景（如"首次销售成交"）
3. 查看推荐的原理组合和参考案例

### 我学了一个原理，想看实际应用

1. 打开 `03-索引/按原理索引.md`
2. 找到原理（如"损失厌恶"）
3. 查看该原理的所有相关案例

### 我想找效果最好的案例参考

1. 打开 `03-索引/按效果索引.md`
2. 按效果排序浏览案例
3. 学习高ROI案例的原理组合

## 理论-案例闭环

```
┌─────────────────────────────────────────────────────┐
│                                                     │
│   理论库                    案例库                   │
│   ┌─────────┐              ┌─────────┐             │
│   │ 原理A   │─────────────→│ 案例1   │             │
│   │         │              │ 案例2   │             │
│   └─────────┘              └─────────┘             │
│        ↑                        │                  │
│        │                        │                  │
│        └────────验证与反哺───────┘                  │
│                                                     │
└─────────────────────────────────────────────────────┘
```

- **理论 → 案例**：每个原理链接到验证该原理的案例
- **案例 → 理论**：每个案例标注使用的底层原理
- **双向验证**：原理指导实践，案例验证原理

## 处理标准

### 理论文档
- 不是"怎么做"，而是"为什么这样做有效"
- 不是话术模板，而是心理机制
- 不是操作步骤，而是设计哲学

### 案例文档
- 必须关联到具体原理
- 必须有效果对比数据
- 必须说明适用边界

### 质量检验
每篇文档问自己：
1. 读完能理解"为什么"吗？
2. 原理能迁移到其他场景吗？
3. 三年后这个知识还有用吗？
